# lab-5
Задание 1. 
Генератор случайных чисел в заданном диапазоне. Не используйте готовые реализации ГПСЧ. Отфильтруйте вывод генератора, чтобы он не содержал чисел с числом делителей больше n.


Отчет: 
Реализация ГПСЧ
Линейный конгруэнтный генератор (LCG)

Использован классический линейный конгруэнтный метод со следующими параметрами:

Модуль (m): 2³¹ - 1 (2147483647) - простое число Мерсенна

Множитель (a): 1103515245

Приращение (c): 12345

Формула: Xₙ₊₁ = (a·Xₙ + c) mod m

Выбор этих параметров обеспечивает:

Максимальный период (m) при правильном выборе a, c, m

Хорошие статистические свойства

Эффективную реализацию

Инициализация
В качестве начального seed используется текущее время в миллисекундах, что обеспечивает различные последовательности при каждом запуске.
Обработка особых случаев
Для числа 0 возвращается бесконечность (чтобы отфильтровать его)
Обрабатываются отрицательные числа (делители считаются по модулю)

Алгоритм фильтрации
Генерируется случайное число в заданном диапазоне
Вычисляется количество его делителей
Если количество делителей ≤ n, число добавляется в результат
Процесс повторяется до получения требуемого количества чисел
Защита от бесконечного цикла: максимальное количество попыток ограничено (count × 100).
Анализ эффективности

Временная сложность
Генерация числа: O(1)
Подсчет делителей: O(√max_val)
Общая сложность для получения k чисел: O(k · √max_val · attempts_factor)
Пространственная сложность
O(1) дополнительной памяти (не считая хранения результатов)

Вероятностные характеристики
Вероятность успешной генерации за одну попытку зависит от:
Диапазона чисел
Порогового значения n
Например, в диапазоне [1, 100]:
Для n=2 (простые числа): ~25% чисел подходят
Для n=4: ~45% чисел подходят
Для n=10: ~75% чисел подходят

Возможные улучшения
Кэширование результатов подсчета делителей для повторяющихся чисел
Использование решета Эратосфена для предварительного вычисления количества делителей для всего диапазона
Адаптивный генератор с изменением диапазона для повышения эффективности
Аппаратная энтропия для улучшения случайности (чтение из /dev/urandom)

Заключение
Разработанный генератор успешно решает поставленную задачу:
Генерирует случайные числа с помощью собственной реализации LCG
Эффективно подсчитывает количество делителей (O(√n))
Фильтрует выходные значения согласно заданному критерию
Обеспечивает защиту от зацикливания при невозможности найти подходящие числа
